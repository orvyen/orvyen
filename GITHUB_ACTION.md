# Orvyen GitHub Action

Automatically audit your SQL/dbt codebase on every pull request and get findings posted as PR comments.

## Features

- ✅ Runs on PR creation/update
- ✅ Posts findings as PR comment
- 🔴 Fails workflow on critical findings
- 📊 Outputs metrics for CI/CD pipelines
- ⚡ Fast execution (< 5 seconds for most projects)

## Setup

### 1. Add Workflow File

Create `.github/workflows/orvyen-audit.yml`:

```yaml
name: Orvyen SQL Audit

on:
  pull_request:
    paths:
      - '**.sql'
      - '**/schema.yml'
      - 'models/**'

jobs:
  audit:
    runs-on: ubuntu-latest
    
    steps:
      - name: Checkout code
        uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Run Orvyen Audit
        id: audit
        uses: orvy/orvyen@v1
        with:
          models-path: models
          github-token: ${{ secrets.GITHUB_TOKEN }}
      
      - name: Fail on critical findings
        if: steps.audit.outputs.critical-count > 0
        run: exit 1
```

### 2. Configure (Optional)

Customize the action with inputs:

```yaml
with:
  models-path: models              # Path to SQL/dbt models (default: models)
  format: json                     # Output format (default: json)
  github-token: ${{ secrets.GITHUB_TOKEN }}  # For posting PR comments
```

## How It Works

1. **Trigger**: Workflow runs on PR that modifies SQL files or schema
2. **Audit**: Orvyen analyzes your models for architectural issues
3. **Comment**: Posts findings summary as PR comment
4. **Fail**: Exits with code 1 if critical findings detected (blocks merge)

## Outputs

The action exposes these outputs for downstream steps:

```yaml
findings-count:    # Total number of findings
critical-count:    # Number of critical findings
report-path:       # Location of generated HTML report
```

### Example Usage

```yaml
- name: Slack notification
  if: steps.audit.outputs.critical-count > 0
  run: |
    curl -X POST ${{ secrets.SLACK_WEBHOOK }} \
      -d "{\"text\": \"⚠️ Critical SQL issues in PR: ${{ steps.audit.outputs.critical-count }} critical findings\"}"
```

## Skip Workflow for Bot PRs

Avoid running on dependabot or other bots:

```yaml
if: github.actor != 'dependabot[bot]'
```

## Troubleshooting

### Action not posting comments
- Ensure `models-path` points to correct directory
- Check that `github-token` is set (defaults to `secrets.GITHUB_TOKEN`)
- Verify workflow has `pull_request` trigger

### Workflow fails unexpectedly
- Run locally: `npx orvyen ./models --format json`
- Check GitHub Actions logs for error details
- Ensure Node.js 20+ is configured

### False positives
- Update `.orvyen.config.js` to customize check severity
- Some warnings can be safely ignored (low severity)

## Configuration File

Create `.orvyen.config.js` to customize per-repo:

```js
module.exports = {
  include: ['models/**/*.sql'],
  exclude: ['models/staging/**'],
  checks: {
    unused_model: true,
    missing_tests: true,
    circular_dependency: true,
    broken_ref: 'critical',  // Override severity
    grain_misalignment: false, // Disable check
  },
  output: 'terminal',
  outputDir: '.orvyen',
}
```

## Advanced: Matrix Testing

Test multiple dbt projects:

```yaml
strategy:
  matrix:
    project:
      - { name: 'Analytics', path: 'analytics/models' }
      - { name: 'Warehouse', path: 'warehouse/models' }

steps:
  - name: Audit ${{ matrix.project.name }}
    uses: orvy/orvyen@v1
    with:
      models-path: ${{ matrix.project.path }}
```
