## How It Works in GitHub

1. **Trigger**: PR modifies `.sql`, `schema.yml`, or `models/` files
2. **Checkout**: GitHub checks out the code
3. **Setup**: Node.js 20 installed
4. **Audit**: Action runs `npx orvyen ./models --format json`
5. **Parse**: Extracts JSON and formats for PR comment
6. **Comment**: Posts summary to PR (optional, skipped if not a PR)
7. **Block**: Exits with code 1 if critical findings (blocks merge)

## Usage

### 1. Copy Workflow File

Add to your repo:

```bash
cp .github/workflows/orvyen-audit.yml your-repo/.github/workflows/
```

### 2. Customize (Optional)

Edit the workflow to match your project:

```yaml
with:
  models-path: dbt/models  # Change if not "models"
  github-token: ${{ secrets.GITHUB_TOKEN }}
```

### 3. Commit & Push

```bash
git add .github/workflows/orvyen-audit.yml
git commit -m "Add Orvyen SQL audit action"
git push
```

### 4. Create a PR

Create any PR that modifies SQL files. The action will:

- Run automatically
- Post findings as comment
- Block merge if critical findings
- Export metrics for CI/CD

## Action Outputs

The action sets these GitHub Actions outputs for downstream steps:

```yaml
findings-count:    # Total findings (e.g., "20")
critical-count:    # Critical findings (e.g., "3")

```

Example downstream usage:

```yaml
- name: Notify Slack on critical
  if: steps.audit.outputs.critical-count > 0
  run: curl ... # Send message to Slack
```

## Troubleshooting

### Action not running?
- Ensure workflow file is in `.github/workflows/`
- Check PR modifies `.sql` or `schema.yml` files
- View GitHub Actions tab for logs

### No JSON output?
- Run locally: `npx orvyen ./models --format json`
- Check your models path matches workflow config

### Critical findings too aggressive?
- Create `.orvyen.config.js` to customize severity
- Set `grain_misalignment: false` to disable
- Set `circular_dependency: 'high'` to downgrade

## Configuration Example

Create `.orvyen.config.js` in repo root:

```js
module.exports = {
  include: ['models/**/*.sql'],
  exclude: ['models/staging/**'],
  checks: {
    unused_model: true,
    missing_tests: true,
    circular_dependency: true,
    broken_ref: 'critical',
    grain_misalignment: false,  // Disabled
  },
}
```
